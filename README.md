# Monitoramento de Rodovias - Sprint 1

## Descrição

Projeto desenvolvido para modelar trechos de rodovia e monitorar o crescimento da vegetação, permitindo priorizar a manutenção.

## Conceitos aplicados

* Classes e Objetos
* Métodos e Comportamentos
* Encapsulamento
* Associação entre classes
* Clean Code

## Perguntas de Reflexão

### 1. Classe vs Objeto

TrechoRodovia é uma classe pois representa um modelo genérico.
Já "BR-116 KM 10 ao 15" é um objeto, pois é uma instância com valores reais.

### 2. Método vs Função

Um método pertence a um objeto e pode alterar seu estado, enquanto uma função é independente.

### 3. Encapsulamento

Se o nível de vegetação fosse público, poderiam ser atribuídos valores inválidos, comprometendo o sistema.
